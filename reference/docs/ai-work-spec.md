# AgentHub AI Work Spec

## 长期目标

把 AgentHub 建成一个以即时聊天为交互方式的多 Agent 任务平台：用户像使用主流 IM 软件一样管理群聊、联系人和聊天记录，同时由 Orchestrator 调用 Claude Code、Codex 或用户自建 Agent 完成真实任务。

## 协作 Skills

- 需求规格化：把自然语言和截图标注转成数据模型、接口、交互和验收项。
- 仓库理解：先阅读 README、产品文档、技术文档、相关源码和已有修改。
- 全栈实现：在 React、FastAPI、SQLite 和 Agent Adapter 边界内完成闭环。
- Agent 集成：检查真实 CLI/API 的路径、参数、认证、超时和错误反馈。
- 体验验证：运行单元测试、类型检查、生产构建和浏览器交互检查。
- 变更记录：每轮同步更新本文件和 `docs/ai-collaboration-log.md`。

## 当前迭代：2026-06-10 全成员调度与细粒度流式输出

### 用户指令整理

1. 群聊主 Agent 必须能够调度当前会话中的所有 Agent，特别是用户使用 `@all` 时，不能把多个任务重复分给少数 Agent。
2. Agent 回复需要更细粒度地流式显示，不能等完整回答或完整句子生成后才一次性展示。

### 本轮工作范围

- 调整 Orchestrator 的计划校正、重复任务合并、能力路由和 `@all` 成员覆盖逻辑。
- 调整 CLI/API 输出分片、消息持久化和 SSE 推送频率。
- 增加后端测试，验证全部成员参与及多次流式更新。
- 运行后端测试、前端构建，并在浏览器检查实时消息更新。

不修改联系人与会话数据模型，不删除聊天记录、项目产物、`.venv/` 或 `node_modules/`。

### 验收标准

- `@all` 任务计划中每个非主 Agent 至少出现一次，同一 Agent 的重复任务会合并。
- 文件能力路由后仍保证所有适合参与的会话成员获得任务。
- DeepSeek、Claude Code 的增量内容以短片段持续更新；只提供最终正文的 CLI 也会渐进写入同一条消息。
- SSE 能及时推送连续内容变化，前端保持同一消息气泡更新。
- 后端测试和前端生产构建通过。
