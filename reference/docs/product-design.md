# AgentHub 产品设计文档

## 目标用户

AgentHub 面向需要同时调用多个 AI 编程或内容 Agent 的开发者、产品经理和小团队。用户希望像使用 IM 一样创建单聊或群聊，并在同一个上下文中完成需求拆解、代码实现、预览验证和文档产出。

## 核心场景

1. 用户创建群聊后，默认只有 System Agent，并可从联系人继续添加 Claude Code、Codex 等 Agent。
2. 用户通过 `@all` 发起复杂任务，当前主 Agent 自动拆解并顺序分派。
3. 多个 Agent 基于同一聊天历史依次回复，产出代码、Diff、网页预览和文档卡片。
4. 用户在聊天流中接受或拒绝 Diff，展开网页预览或代码块，继续追问局部修改。
5. 用户把关键消息置顶为长期上下文，后续 Agent 回复会围绕这些信息继续迭代。

## 信息架构

- 左侧：群组、联系人和归档会话。群组展示群聊；联系人可直接进入对应 Agent 的单聊，并可移除用户自建联系人；归档会话只读。
- 中间：聊天流，支持单聊、群聊、消息操作和产物内联。
- 右侧：仅展示当前会话的 Agent 成员；群聊提供“+”从联系人添加成员，并保留“创建新 Agent”入口。
- 弹层：产物展开预览，用于网页 iframe、代码、Diff、文档和演示文稿。
- 自建 Agent 弹层：填写名称、功能说明、System Prompt，并选择仅当前会话或同时保存到联系人。
- 成员详情弹层：单聊只读展示名称与 System Prompt；群聊可编辑会话级名称、职责和 System Prompt，并切换主 Agent。

## 关键交互

- `@Agent`：指定某个 Agent 回复。
- `@all`：触发群聊协作；主 Agent 在计划消息中按执行顺序逐项 `@Agent`，后续 Agent 依次回复。
- 普通消息：单聊由唯一 Agent 独立完成；群聊由当前主 Agent 理解需求、拆解子任务并顺序调用成员。
- 主 Agent：默认是 System Agent。切换主 Agent 时，新旧主 Agent 交换当前会话中的职责和 System Prompt，不修改联系人默认值。
- 执行上下文：每个 Agent 开始工作前读取完整历史、置顶消息和自己的会话级 System Prompt。
- 项目产物：主 Agent 必须声明当前会话的唯一项目目录；代码、文本、规范和协作文档都只能写入该目录。
- 能力边界：Claude Code、Codex 等 CLI Agent 可操作项目文件；DeepSeek API Agent 负责分析、研究和文本任务，其文字成果自动归档到会话项目目录。
- Accept/Decline：对 Diff 产物标记状态，为后续真实应用补齐入口。
- 置顶消息：把关键约束沉淀为长期上下文。
- 会话管理：标题可编辑，支持置顶、归档和永久删除；归档后保留历史但禁止继续发送。
- 自建 Agent：创建后加入当前会话；保存到联系人时可在后续从联系人发起单聊。
- 删除联系人：仅移除用户自建 Agent 的联系人入口，不删除历史聊天或已有群聊成员；三个系统预设 Agent 不可删除。
